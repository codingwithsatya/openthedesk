Week 1 complete — 8 / 8 milestones
Core LLM concepts
Tokens
~¾ of a word. Unit of cost and context. Claude = 200K token window. You pay per input + output token.
Embeddings
Meaning encoded as numbers (vectors). Similar meaning = similar vectors. Foundation of RAG in Month 2.
Temperature
0 = deterministic. 0.5 = balanced. 1.0 = creative. Use low (0–0.3) for structured/JSON output.
Stateless API
No memory between calls. You send full conversation history every single time. Managing this = key engineering challenge.
System vs user prompt
System = persistent rules (role, format, constraints). User = actual input. Put constraints in system prompt.
Prompt engineering patterns

Prompt structure

Role + Context + Task + Format. Always define all four layers for reliable, consistent output.

Chain-of-thought

Add "think step by step" before complex tasks. Dramatically improves accuracy on reasoning problems.

Few-shot learning

Show 2–3 input/output examples. Model matches the pattern without needing long instructions.

Structured output

"Respond with valid JSON only. No markdown. No backticks." Then use json.loads() to parse.

Edge case handling

Always define what to return when data is missing. "If no phone number, return null." Else model hallucinates.

Prompts are code

Test on 50 inputs. Version control them. Handle errors. Treat prompts exactly like software functions.

Conversation memory

Streaming responses

Week 3 complete — 8 / 8 milestones
Core concepts
Persona + Constraint + Format trifecta
The most reliable system prompt structure. Persona sets expertise, constraints define what the model won't do, format locks the output shape. All three together = consistent production output.
Structured output as a contract
Say it 3 ways: "ONLY valid JSON. No prose. No markdown. No backticks." Give the exact schema. Define the null case explicitly ("if no issues, return []"). Missing any one of these causes KeyErrors in prod.
XML tags for complex inputs
Wrap user-submitted code in <code> tags. Prevents the model confusing instructions with content. Especially important when code contains prompt-like strings.
Temperature 0.1 for structured output
Use near-zero temperature when parsing JSON. Creative temperature randomly changes schema shape. 0.1 gives near-deterministic output while avoiding the occasional freeze of 0.
Prompts as functions — engineering mindset
Function signature = input schema. Return type = JSON shape. Error handling = fallback instructions. Unit tests = 20–30 fixed inputs. Version control prompts in git like code.
Global env var override pitfall
load_dotenv() only sets a variable if it's NOT already in the environment. A stale global key in ~/.zshrc silently wins over .env. Fix: unset ANTHROPIC_API_KEY in terminal, or remove from shell config.

Core concepts
Supabase pgvector setup
Enable vector extension. Create documents + chunks tables. Build match_chunks() SQL function with cosine distance operator <=>. user_id filter for data isolation.
Voyage AI embeddings
voyage.embed(texts, model="voyage-3", input_type="document"). Returns 1024-dim vectors. input_type="query" for search queries. Batch up to 128 texts per call. Free tier = 3 RPM.
PDF text extraction
PyPDF2.PdfReader. Extract page by page. Some PDFs have no text layer (scanned) — PyPDF2 returns empty. Text-only RAG works for 80% of real documents.
Chunking with overlap
Split by word count (CHUNK_SIZE=400, CHUNK_OVERLAP=50). Overlap prevents key sentences being split across chunks. Skip chunks under 50 chars — they're noise.
Ingestion pipeline
PDF → extract pages → chunk text → embed chunks (Voyage) → store in Supabase with doc_id, user_id, page_number, chunk_index. Insert in batches of 50.
Retrieval pipeline
Embed query (Voyage, input_type="query") → supabase.rpc("match_chunks") → get top 5 chunks → build context string → Claude answers with citation constraint.
Confidence threshold
match_threshold=0.3 in match_chunks(). If 0 chunks returned, respond "I don't have that information." Never let Claude answer from empty context — it hallucinates.
File upload endpoint
FastAPI UploadFile + python-multipart. Save to tempfile, ingest, delete. Return doc_id, pages, chunks count. /ask endpoint embeds question + retrieves + asks Claude.
Code patterns
Supabase vector search function
-- SQL function in Supabase create or replace function match_chunks( query_embedding vector(1024), match_user_id text, match_count int default 5, match_threshold float default 0.5 ) returns table (id uuid, content text, page_number int, similarity float) as $$ select id, content, page_number, 1 - (embedding <=> query_embedding) as similarity from chunks where user_id = match_user_id and 1 - (embedding <=> query_embedding) > match_threshold order by embedding <=> query_embedding limit match_count; $$ language sql stable;

Week 4 complete — 7 / 7 milestones · Month 1 complete 🎉
Core concepts
Prompt caching
Add cache_control: ephemeral to your system prompt. Claude caches it after the first call. Subsequent calls cost 90% less on those tokens. One line change — biggest cost lever in production.
Batch processing
Submit multiple inputs in one API call. Returns individual results + cross-file summary. The summary object (avg score, total issues, highest risk) is what dashboards and managers actually read.
Model routing by task complexity
Haiku = 10x cheaper for simple classification. Sonnet = best cost/quality for reasoning. Opus = complex multi-step agents. Routing cheap tasks to Haiku cuts your bill 60–70% at scale.
Cost math at scale
10K calls/day × 500 token system prompt = 5M input tokens/day on system prompt alone. With caching: ~500K tokens/day. That's the difference between $8/day and $0.80/day just on the system prompt.
Component decomposition in React
Split SingleAnalyzer, BatchAnalyzer, and ResultCard into separate components. ResultCard is shared between both. Keeps the file readable and prevents duplicating the severity card + suggestions logic.

Week 7 complete — 6 / 6 milestones · Advanced RAG pipeline shipped 🎉
Core concepts
Hybrid search — semantic + keyword combined
Semantic search finds meaning. Keyword search (BM25) finds exact terms. Hybrid combines both with configurable weights (50/50 for OpenTheDesk). Fixes the PTR-FAST vs PTR-FULL confusion problem.
BM25 keyword search
Best Match 25 algorithm — counts word frequency weighted by document length. rank_bm25 library in Python. Tokenize corpus once, score all docs against query. Returns 0 for no match (expected for conceptual queries).
Reciprocal Rank Fusion (RRF)
Merges two ranked lists into one. Formula: score += 1/(60 + rank) for each list. Items ranked high in both lists bubble to top. k=60 constant prevents extreme ranks from dominating. No score normalization needed.
Reranking with Cohere
After hybrid search returns 20 candidates, Cohere rerank-english-v3.0 reads query + each chunk together and scores true relevance. 0.99 = extremely confident. ~$1/1000 queries. Use top_n=5 from 20 candidates.
Confidence threshold
Check rerank_score of top result. Below 0.1 = refuse to answer. Above 0.65 = answer confidently. Prevents confident wrong answers from irrelevant chunks. Critical for OpenTheDesk — wrong rule during live trade = bad trade.
Python-side hybrid search (free tier workaround)
Supabase free tier blocks GIN indexes (memory limit 32MB). Solution: semantic search via pgvector, BM25 keyword search in Python over retrieved chunks, RRF merge in Python. Cleaner architecture anyway — more control.
Code patterns
Full hybrid search pipeline
def hybrid_search_pipeline(question, top_k=5): # 1. Embed question vector = voyage.embed([question], model="voyage-3", input_type="query") # 2. Semantic search — top 20 from pgvector semantic = semantic_search(vector, top_k=20) # 3. BM25 keyword search over same chunks keyword = keyword_search(question, semantic, top_k=20) # 4. RRF merge fused = reciprocal_rank_fusion(semantic, keyword) # 5. Rerank top candidates return rerank(question, fused[:20], top_n=top_k)
BM25 keyword search in Python
from rank_bm25 import BM25Okapi def keyword_search(query, chunks, top_k=20): tokenized = [c["content"].lower().split() for c in chunks] bm25 = BM25Okapi(tokenized) scores = bm25.get_scores(query.lower().split()) ranked = sorted(enumerate(scores), key=lambda x: x[1], reverse=True) return [chunks[i] for i, score in ranked[:top_k] if score > 0]
Cohere reranking
import cohere co = cohere.Client(api_key=os.environ["COHERE_API_KEY"]) def rerank(query, chunks, top_n=5): docs = [c["content"] for c in chunks] response = co.rerank( model="rerank-english-v3.0", query=query, documents=docs, top_n=top_n ) return [chunks[r.index] | {"rerank_score": r.relevance_score} for r in response.results]

Google Doc retry pattern

Ribbon vision — cloud vs candle color

Single data source architecture

Saty ATR calculation — exact formula
ATR is NOT PDH-PDL (71.03 today). It's a smoothed value from Saty's custom indicator. Back-calculated: upper trigger - PDC / 0.236 = ATR. Today: (7418.62 - 7400.97) / 0.236 = 74.79. Must enter manually each morning until formula confirmed.
11 Saty ATR levels (not 8)
Added missing 50% level on both sides. Full set: Full ATR ↑, GG Complete ↑, 50% ↑, GG Open ↑, Call Trigger, PDC, Put Trigger, GG Open ↓, 50% ↓, GG Complete ↓, Full ATR ↓. All shown to 2dp precision.
ATR-based strike targeting (not delta)
Tradier greeks unreliable intraday for SPX index options — delta=1.0 across most strikes. Per Saty system: target 0.5 ATR OTM instead. best_strike() finds closest real-priced strike to that target. Delta shown when available.
Options chain — after hours limitation
After market close, OTM options go to near-zero mid, greeks go stale. Chain only useful 9:35–4pm ET. Real test tomorrow morning — chain should show proper pricing and greeks across all strikes.
Saty ATR = ATR-14 Wilder (confirmed from Pine Script source)
Pine Script: ta.atr(14) with session=session.extended. Formula: TR = max(H-L, |H-prevClose|, |L-prevClose|). Seed = avg of first 14 TRs. Then: ATR = (prev_ATR × 13 + today_TR) / 14. We get 75.79 vs Saty's 74.79 — 1pt delta from extended session ticks (pre/after hours). Tradier history = regular session only. Permanent ~1pt gap, close enough for trading.
Trading mode support — Day / Multiday / Swing
Day = Daily ATR-14. Multiday = Weekly ATR-14. Swing = Monthly ATR-14. /market-data accepts ?trading_mode=day|multiday query param. Pine Script maps: Day→D, Multiday→W, Swing→M, Position→3M, Long-term→12M. Same Wilder formula, different timeframe bars.

ATR-14 Wilder implementation
def _wilder_atr(bars, period=14): trs = [] for i in range(1, len(bars)): h, l, pc = bars[i]["high"], bars[i]["low"], bars[i-1]["close"] trs.append(max(h - l, abs(h - pc), abs(l - pc))) # Seed with simple average of first 14 TRs atr = sum(trs[:period]) / period # Wilder smoothing — same as Pine ta.atr(14) for tr in trs[period:]: atr = (atr * (period - 1) + tr) / period return round(atr, 2)
